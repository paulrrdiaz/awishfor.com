## Why

Site ships placeholder favicon (create-t3-app default `/favicon.ico`, `MyWebSite`/`MySite` manifest strings). No branded icon set for A Wish For — hurts polish across browser tabs, bookmarks, iOS home screen, and Android/PWA install.

## What Changes

- Replace `public/favicon.ico` and add full icon set (favicon.ico, favicon.svg, favicon-96x96.png, apple-touch-icon.png, web-app-manifest-192x192.png, web-app-manifest-512x512.png) sourced from `/Users/paulrrdiaz/Downloads/favicon`.
- Add `public/site.webmanifest`, updated with real app name (`A Wish For` / short name `A Wish For`) instead of generator placeholders (`MyWebSite`/`MySite`).
- Update `src/app/layout.tsx` `metadata` to reference the full icon set (icons array + apple touch icon) and register the web manifest, replacing the single `/favicon.ico` link.
- Update root `metadata.title`/`description` placeholders (`Create T3 App` / `Generated by create-t3-app`) to real site copy, since they ship alongside the favicon in the same metadata block.

## Capabilities

### New Capabilities
- `site-favicon`: branded favicon/icon set and web manifest served for the site, wired into root metadata.

(none — no existing capability spec covers site icons/metadata)

- Affected files: `public/favicon.ico` (replace), `public/site.webmanifest` (new), `public/favicon.svg`, `public/favicon-96x96.png`, `public/apple-touch-icon.png`, `public/web-app-manifest-192x192.png`, `public/web-app-manifest-512x512.png` (new), `src/app/layout.tsx` (metadata update).
